### System Improvement Suggestion

> **The post workflow has a broken handoff: drafts live in this repo, but publishing happens in a browser tab that this bot never touches.** The result is three consecutive days of warnings with zero action. The fix is a closed-loop protocol: every post file must contain a checklist at the bottom — `[ ] Draft committed`, `[ ] Posted/Scheduled`, `[ ] Engagement checked at 2h`. This bot should refuse to mark a post "done" in the backlog until the file shows `[x] Posted` with a timestamp. Without a confirmed `[x]`, the task stays on the daily log every single day until it is checked. That is the only mechanic that closes the loop — treat an unchecked `[ ] Posted` the same way a developer treats a failing CI check: it is not done until it is green.
